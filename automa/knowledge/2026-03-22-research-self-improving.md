Data Flywheel\n\n**What it is**: User interactions generate training data automatically. The cycle: user asks question, AI answers, user behavior signals quality (implicit feedback), good examples get added to training data, model improves, better answers generate more positive signals.\n\n**Implicit feedback signals for a tutor**:\n- **Positive**: Student completes experiment after AI guidance. Student asks follow-up questions (engagement). Student does not rephrase the same question (understood first time). Student clicks \"ask teacher\" or external help after AI response. Student sends \"non capisco\" or similar frustration signals.\n\n**Pipeline architecture**:\n```\nUser interaction → Log (question + response + context + outcome)\n                 → Implicit scoring (heuristic rules on signals above)\n                 → Filter (score > threshold)\n                 → Human review queue (sample)\n                 → Approved examples → Training dataset\n                 → Periodic fine-tune → Deploy\n```\n\n**Critical insight**: The flywheel does NOT mean \"automatically train on all user data.\" That path leads to model collapse and amplified errors.
