Physical-Digital Gap (Simulation vs Real Hardware)\n\nBased on the [PLOS ONE meta-analysis (Dec 2024)](https://journals.plos.org/plosone/article?id=10.1371/journal.pone.0316269), [Tselegkaridis et al. 2024](https://www.mdpi.com/2078-2489/15/11/672), and [ScienceDirect 2025 VR study](https://www.sciencedirect.com/science/article/pii/S259029112500470X):\n\n**Key findings**:\n- The PLOS ONE meta-analysis of 46 studies found virtual labs have a **positive but moderate effect** on engineering students' performance.\n- For **Arduino and microcontrollers specifically**, Tselegkaridis et al. found that virtual and physical labs contributed **equally** to performance progress -- this is strong validation for ELAB's simulator approach.\n- **Conceptual knowledge** improves the most from virtual labs, followed by inquiry skills. **Measurement/hands-on skills** improve least -- the main weakness of simulation-only approaches.\n- **Hybrid models** (simulation + physical) consistently outperform either alone. Scratch to Arduino Compilation: Technical State of the Art\n\nBased on [S4A](https://s4a.cat/), [mBlock](https://mblock.cc/), [eBlock](https://github.com/distintiva/eBlock), and the [Arduino Forum discussion](https://forum.arduino.cc/t/circuit-simulator-block-programming-arduino-upload-holy-grail/1375874):\n\n**Three architectural approaches**:\n\n| Approach | Tools | How it works | Pros/Cons |\n|----------|-------|-------------|-----------|\n| **Firmata tethered** | S4A, Snap4Arduino | Board runs Firmata firmware. Could enable \"save settings\" experiments.\n- **ADC**: Supported with configurable reference voltage.\n\n**Alternatives**:\n\n| Emulator | Language | Browser? | Speed | Peripherals |\n|----------|----------|----------|-------|-------------|\n| **avr8js** | TypeScript | Native JS | Good | Comprehensive |\n| **SimAVR (Emscripten port)** | C->JS | Yes (via port) | Faster CPU | Less maintained |\n| **Slimavr** | C | No | 38MHz (fastest) | Minimal |\n| **AVR8js + CircuitJS1** | JS + GWT | Yes | Combined | Analog circuit sim |\n\n**WebAssembly opportunity**: avr8js is pure TypeScript/JavaScript. A [WASM compilation of SimAVR](https://github.com/sarthaksarm/Port-of-SimAVR) exists but only supports ATmega1280. The performance gain from WASM would be marginal for ELAB's use case since avr8js running at ~16MHz equivalent is already sufficient for Arduino Uno simulation. The bottleneck is typically rendering, not CPU emulation.\n\n**Expanding peripheral emulation for ELAB**:\n- **EEPROM**: Low effort, already in avr8js. Would enable \"persistent memory\" experiments.\n- **I2C devices**: avr8js has TWI support. Adding virtual I2C device stubs (temperature sensor, OLED) would expand experiment possibilities significantly.\n- **Interrupts**: avr8js supports hardware interrupts. This could enable button interrupt experiments (more advanced).\n- **Analog comparator**: Not in avr8js. Lower priority.\n\n**Actionable for ELAB**: The biggest opportunity is not emulating more AVR peripherals (avr8js already has most) but **creating more virtual peripheral simulations in the frontend** -- virtual ultrasonic sensor, virtual temperature sensor, virtual IR remote -- that communicate with avr8js through the existing GPIO/ADC/I2C interfaces.\n\n---\n\n## Summary of Top Recommendations
